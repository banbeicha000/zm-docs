关于counter ​

上面需要写很多 .slide-enter-content > *:nth-child(1) { --stagger: 1; }，那么是否可以借用counter代替这一步

css
.slide-enter-content {
  counter-reset: enter-count;
}

.slide-enter-content > p {
  counter-increment: enter-count;
  --stagger: counter(enter-count);
  ...
  animation-delay: calc(var(--start) + var(--stagger) * var(--delay));
}

事实证明不行，原因是counter返回的是字符串，并不是数字，而calc就会计算失败。目前没有办法转化为数字。

scroll-snap-type ​

INFO

CSS 属性 scroll-snap-type 设置了在有滚动容器的情形下吸附至吸附点的严格程度。

取值: ​

  1. none 在滚动此滚动容器的可见视口时，必须忽略吸附点。
  2. x 滚动容器仅在其横轴上吸附至吸附位置。
  3. y 滚动容器仅在其纵轴上吸附至吸附位置。
  4. block 滚动容器仅在其块向轴上吸附至吸附位置。
  5. inline 滚动容器仅在其行向轴上吸附至吸附位置。
  6. both 滚动容器在其两轴上独立地吸附至吸附位置（可能在各轴上吸附至不同的元素）。
  7. mandatory 若滚动容器当前未在滚动，则其可见视口必须吸附至吸附位置。
  8. proximity 若滚动容器当前未在滚动，则其可见视口可以吸附至吸附位置。是否吸附由用户代理根据滚动参数决定。若指定了吸附轴，则此为默认的吸附程度。

scroll-snap-align ​

INFO

scroll-snap-align 属性将盒子的吸附位置指定为其吸附区域（作为对齐对象）在其吸附容器的吸附口（作为对齐容器）中的对齐方式。其两值分别指定了在块向轴盒行向轴上的吸附对齐方式。若仅指定一值，则第二值默认为同一值。

取值: ​

  1. none 此盒在此轴上未定义吸附位置。
  2. start 此盒的滚动吸附区域在滚动容器的吸附口中的首对齐为此轴上的吸附位置。
  3. end 此盒的滚动吸附区域在滚动容器的吸附口中的末对齐为此轴上的吸附位置。
  4. center 此盒的滚动吸附区域在滚动容器的吸附口中的居中对齐为此轴上的吸附位置。
