vue2项目构建工具vue-cli迁移到vite ​

INFO

借用vite文档说法：当我们开始构建越来越大型的应用时，需要处理的 JavaScript 代码量也呈指数级增长。 包含数千个模块的大型项目相当普遍。基于 JavaScript 开发的工具就会开始遇到性能瓶颈：通常需要很长时间（甚至是几分钟！）才能启动开发服务器， 即使使用模块热替换（HMR），文件修改后的效果也需要几秒钟才能在浏览器中反映出来。如此循环往复，迟钝的反馈会极大地影响开发者的开发效率和幸福感

对于一些使用vue2的老项目，从零开始重新构建到vite，可能不太现实。结合现有情况，决定采取使用插件解析vue，vue2相关文件保持不变，降低迁移到vite的代价

核心插件 vite-plugin-vue2 (作者antfu) ​

首先有几点说明一下：

  • 提起vite，可能第一印象是vue3，这两者最配。固有印象觉得vue2无法使用vite，实际上，vite核心就是作为一个打包工具，目前支持很多框架react、svelte等，那么脱离来看，支持vue2也不足为奇，只不过对于vue3默认支持更好，而对于支持vue2需要借助插件
  • 借用vite-plugin-vue2该插件的作用是由于 Vite 是基于 ES Module 的开发服务器和构建工具，而 Vue 2.x 是基于 CommonJS 的，因此需要使用该插件来在 Vite 中使用 Vue 2.x
  • 另外vite-plugin-vue2插件只支持Vue 2.6 or earlier，支持vue2.7需要vue2官方插件 @vitejs/plugin-vue2

新增、删除依赖 ​

  1. 删除依赖
json
// 删除 vue-cli 相关依赖
"@vue/cli-plugin-babel": "~4.5.0",
"@vue/cli-plugin-router": "~4.5.0",
"@vue/cli-plugin-vuex": "~4.5.0",
"@vue/cli-service": "~4.5.0"
// 另外，core-js、autoprefixer等也可以删除了
"autoprefixer": "^8.0.0",
"core-js": "^3.6.5",
  1. 新增依赖
shell
yarn vite vite-plugin-vue2 -D
json
"vite": "^5.2.11",
"vite-plugin-vue2": "^2.0.3",

特别说明，vue与vue-template-compiler版本需要一致

json
"vue": "^2.6.11",
"vue-template-compiler": "^2.6.11"
// 上面引入的包版本号会有差异，导致项目迁移成功后，组件库element-ui部分组件运行出问题，需如下锁定一致版本号
"vue": "2.6.11",
"vue-template-compiler": "2.6.11"

配置vite.config ​

项目根目录下新建vite.config.js文件（记得删除原先的vue.config.js）

js
// vite.config.js
import { defineConfig } from 'vite'
import { createVuePlugin } from 'vite-plugin-vue2'

export default defineConfig({
    plugins: [
        createVuePlugin(/* options */),
        gzipPlugin(),
    ]
})

正常来说，配置到这一步基本完成迁移的功能，但是有些细节需要完善

  1. vue2基本使用路径别名@、#等，同样须在配置文件中添加
js
import path from 'path' // es语法，所有require引入需换成import

export default defineConfig({
    plugins: [
        createVuePlugin(/* options */),
        gzipPlugin(),
    ],
    resolve: {
        alias: {
            "@": path.resolve(__dirname, 'src'),
            "#": path.resolve(__dirname, 'src/components'),
        },
        // 下面这种配置更精确，但会影响node_modules，例如element-ui 组件中使用 ‘@/’引入方式，会被错误解析替换，导致渲染出问题
        // alias: [
        //     { find: /^@/, replacement: path.resolve(__dirname, 'src') },
        //     { find: /^#/, replacement: path.resolve(__dirname, 'src/components') }
        // ],
        extensions: ['.mjs', '.js', '.ts', '.jsx', '.tsx', '.json', '.vue'],
    },
})
  1. 一般vue2引入组件习惯省略.vue后缀，在vite中需要明确带入后缀，这个时候要么全局补充引入后缀，要么在配置中重新覆盖 extensions ，具体可以查看vite官方文档说明
js
export default defineConfig({
    ...
    resolve: {
        ...
        extensions: ['.mjs', '.js', '.ts', '.jsx', '.tsx', '.json', '.vue'],
    },
})
  1. 全局样式变量 如使用less定义的全局变量使用（或者scss，具体配置可以查看vite官方文档）
less
// src/styles/variable.less
@fontColor: white;
js
export default defineConfig({
    ...
    resolve: {
        ...
    },
    css: {
        preprocessorOptions: {
            less: {
                // 也可以使用路径别名 `@import "@/styles/public.less";`
                additionalData: `@import "src/styles/public.less";`
            }
        }
    }
})
  1. 服务代理
js
export default defineConfig({
    ...
    resolve: {
        ...
    },
    // 具体可查看官方文档
    server: {
        proxy: {
            ["/api"]: {
                target: "http://10.139.12.202:8000",
                changeOrigin: true,
                // pathRewrite: { "^/api": "" }
            }
        }
    }
})

环境变量 ​

Vite 在一个特殊的 import.meta.env 对象上暴露环境变量，因此 vue2相关的环境变量需要更新，例如

js
// 内置的环境变量
process.env.NODE_ENV >>> import.meta.env.PROD or import.meta.env.MODE
// 自定义的环境变量 .env 命名约定可以保持不变，另外只有以 VITE_ 为前缀的变量才会暴露给经过 vite 处理的代码，具体参考官方文档
VITE_SOME_KEY=123

清除命名动态导入的注释 ​

js
// 清除 /* webpackChunkName: "notFound" */
{
    path: "/404",
    component: () => import(/* webpackChunkName: "notFound" */"../views/404"),
    name: "404",
    meta: {
      title: "404",
    }
},

替换require ​

js
const src = require('@/assets/icon/search/refresh.png')

或者模板中循环动态引入

vue
<img alt="" :src="item.src ? item.src : require(`@/assets/icon/search/${dealSrcTrans(item.class)}`)" />

需要改为直接import导入，对于循环动态请求资源，需要导入全部资源，在模板中import是不支持的

js
import refresh from '@/assets/icon/search/refresh.png'
import search from '@/assets/icon/search/search.png'
// 也可以通过new URL的方式去得到一个完整连接，这种方法类似于path的拼接路径
new URL(`../assets/icon/${name}.png`, import.meta.url).href

移动html与手动引入main.js ​

  1. Vite 将应用程序的 index.html 文件放在项目的根目录而不是公共目录中，因此需要移动它。并且将<%= htmlWebpackPlugin.options.title %>占位符的实例更改为硬编码值
html
<html>
  <head>
    <!--remove-->
    <link rel="icon" href="<%= BASE_URL %>favicon.ico">
    <!--replace-->
    <link rel="icon" href="/favicon.ico">
  </head>
  <body>
      <!--remove-->
      <noscript>
        <strong>We're sorry but <%= htmlWebpackPlugin.options.title %> doesn't work properly without JavaScript enabled. Please enable it to continue.</strong>
      </noscript>
      <!--replace-->
      <noscript>
        <strong>We're sorry but doesn't work properly without JavaScript enabled. Please enable it to continue.</strong>
      </noscript>
  </body>
</html>
  1. 应用程序不再自动引入main.js，需要手动引入, 注意添加type="module"
html
<html>
  ...
  <body>
      <div id="app"></div>
      <script type="module" src="/src/main.js"></script>
  </body>
</html>

require.context()失效 ​

一些vue2 项目喜欢使用 vuex-persistedstate 插件，并使用require.context()自动注册持久化，如下

js
import Vue from 'vue'
import Vuex from 'vuex'
import persistedState from 'vuex-persistedstate'
import getters from './getters'

Vue.use(Vuex)
const modulesFiles = require.context('./modules', true, /\.js$/)
const modules = modulesFiles.keys().reduce((modules, modulePath) => {
    const moduleName = modulePath.replace(/^\.\/(.*)\.\w+$/, '$1')
    const value = modulesFiles(modulePath)
    modules[moduleName] = value.default
    return modules
}, {})

export default new Vuex.Store({
  modules,
  getters,
  //vuex持久化插件
  plugins: [
    persistedState({
      //存在session里
      storage: window.sessionStorage,
      reducer(val) {
        //只持久化特定的state
        return {
          userInfo: val.user.userInfo
        }
      },
    }),
  ],
})

vite提供了替代方法 import.meta.glob

js
const modulesFiles = import.meta.glob('./modules/*.js', { eager: true })
const modules = Object.keys(modulesFiles).reduce((modules, modulePath) => {
    // 替换modules是因为开启了命名空间 namespaced: true
    const moduleName = modulePath.replace(/(.\/modules\/)|(.js)/g, '') 
    const value = modulesFiles[modulePath]
    modules[moduleName] = value.default
    return modules
}, {})

注意事项 ​

  1. 代理配置问题
    在webpack中，一般很多接口会以api作为前缀，但是在vite中，如果文件命名也包含api，就会在访问本地文件的时候被代理走，从而找不到文件导致404.所以在配置proxy需要注意不要和本地文件冲突（vite的加载文件方式都是通过直接加载本地文件来实现的，每一次的加载都是网络请求）
  2. element-ui按需引入失效
    迁移后，element-ui库的babel-plugin-component按需引入方式不再支持
  3. 编辑器未高亮定位路径别名
    路径别名配置正确，但是编辑器未正确识别定位（如webpack）, 可在项目根目录手动配置 jsconfig.json
json
// jsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],
      "#/*": ["src/components/*"]
    }
  },
  "exclude": ["node_modules"]
}
