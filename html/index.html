Element.toggleAttribute() ​

Element 接口的 toggleAttribute() 方法切换给定元素的某个布尔值属性的状态（如果属性不存在则添加属性，属性存在则移除属性）

Details

Element.toggleAttribute(name [, force])
参数
name
需要切换显示的布尔属性名称，可以是自定义的规范中没有的属性。例如：
 document.body.toggleAttribute('xxx')
force
 布尔值。true的话就是强制设置该属性为true，也就是添加该属性；false的话则表示移除该布尔属性
返回值
如果toggleAttribute()方法执行是添加属性，则返回值值是true，否则就是false

html
<input id="input"> <button id="button" value="启用">禁用</button>
css
/* 按钮文字变化 */
:disabled ~ button {
    -webkit-text-fill-color: transparent;
}
:disabled ~ button::before {
    position: absolute;
    content: attr(value);
    -webkit-text-fill-color: currentColor;
}
js
button.addEventListener('click', _ => {
    input.toggleAttribute('disabled');
})

popover ​

全局属性 popover 用来指定一个元素为弹出式元素（popover element）

说明
  • 弹出式元素通过 display: none 被隐藏，直到通过调用/控制元素（即带有 popovertarget 属性的 <button> 或 <input type="button">）或 HTMLElement.showPopover() 调用打开
  • 当打开时，弹出式元素将出现在所有其他元素之上，即在顶层上，并且不会受到父元素的 position 或 overflow 样式的影响
  • popover 属性可以有 "auto" (en-US)（默认）或 "manual" 的取值

两种不同的方式创建弹出框

  • 以声明方式。可以使用以下代码创建带有切换按钮的简单弹出窗口
html
<button popovertarget="my-popover">打开 Popover</button>

<div popover id="my-popover">Popover 出现</div>

Popover 出现
html
<button popovertarget="mypopover" popovertargetaction="show">
  Show popover
</button>
<button popovertarget="mypopover" popovertargetaction="hide">
  Hide popover
</button>
<div id="mypopover" popover>Popover content</div>
  • 通过 JavaScript API。例如，HTMLElement.togglePopover（） 可用于在显示和隐藏之间切换弹出框
html
<button id="toggleBtn">Toggle popover help UI</button>

<p id="keyboard-help-para">Click the above button or press "h" to get help with using our awesome app.</p>

<div id="mypopover">
  <h2>Help!</h2>

  <p>You can use the following commands to control the app</p>

  <ul>
    <li>Press <ins>C</ins> to order cheese</li>
    <li>Press <ins>T</ins> to order tofu</li>
    <li>Press <ins>B</ins> to order bacon</li>
    <hr />

    <li>Say "Escape" to engage the ejector seat</li>

js
function supportsPopover() {
  return HTMLElement.prototype.hasOwnProperty("popover")
}

const popover = document.getElementById("mypopover")
const toggleBtn = document.getElementById("toggleBtn")

const keyboardHelpPara = document.getElementById("keyboard-help-para")

const popoverSupported = supportsPopover()

if (popoverSupported) {
  popover.popover = "auto"
  toggleBtn.popoverTargetElement = popover
  toggleBtn.popoverTargetAction = "toggle"

  document.addEventListener("keydown", (event) => {
    if (event.key === "h") {
      popover.togglePopover()
    }
  })
} else {
  toggleBtn.style.display = "none"
  keyboardHelpPara.style.display = "none"
}

HTML form validation ​

validity

Each form element (input, for example) provides a validity property which represents a ValidityState. ValidityState looks something like this:

js
{ 
    badInput: false
    customError: true
    patternMismatch: false
    rangeOverflow: false
    rangeUnderflow: false
    stepMismatch: false
    tooLong: false
    tooShort: false
    typeMismatch: false
    valid: false
    valueMissing: true
}

Each property in the ValidityState can roughly match a specific validation issue: valueMissing would match the required attribute, tooLong and tooShort match minLength and maxLength, etc

js
const inputEl = document.querySelector('#input')
if (inputEl.validity.valueMissing) {
    // Simply setting the message by setCustomValidity doesn't show the message, however
    inputEl.setCustomValidity('This is required!') 
    // To get the error to display to the user, use the form element's reportValidity method
    inputEl.reportValidity()
} else {
    // Clear any previous error
    inputEl.setCustomValidity('')
}
